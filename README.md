# procko.pro — Tyler T. Procko, Ph.D.

Personal portfolio and CV site for [Tyler T. Procko](https://procko.pro) — AI/Knowledge Graph Engineer, Ontologist, DoD-cleared.

## Architecture

One source of truth: **`public/data/tyler-procko.yaml`**. All pages read from it at runtime; nothing is hardcoded into components.

```
public/data/tyler-procko.yaml   ← single source of truth for all CV/portfolio content
│
├── .github/
│   ├── workflows/
│   │   ├── deploy.yml                        ← build + deploy to GitHub Pages on push to main
│   │   ├── sync-repo-descriptions.yml        ← weekly sync of GitHub project descriptions
│   │   └── sync-openalex-publications.yml    ← weekly sync of new publications from OpenAlex
│   └── scripts/
│       └── sync_openalex_publications.py     ← queries OpenAlex by ORCID, deduplicates by DOI + title,
│                                                updates cited_by_count, appends new stubs
│
├── build.py                    ← Python/Jinja2 builder → public/legacy/index.html
│   templates/                  ← Jinja2 partials (one per section)
│
├── public/
│   ├── legacy/index.html       ← legacy HTML CV (generated by build.py, served at /legacy)
│   ├── data/tyler-procko.yaml  ← static asset fetched by the React app at runtime
│   ├── CNAME                   ← custom domain (procko.pro)
│   └── 404.html                ← SPA routing redirect for GitHub Pages
│
└── src/
    ├── lib/
    │   ├── yaml-loader.ts      ← fetches + validates YAML at runtime (Zod)
    │   ├── schema.ts           ← Zod schema matching the YAML structure
    │   ├── cv-data.ts          ← transforms Person → renderer-agnostic CVData/ResumeData models
    │   ├── graph-builder.ts    ← builds Cytoscape node/edge graph from Person data
    │   ├── tech-categories.ts  ← maps technology/skill names to graph categories
    │   └── ui-constants.ts     ← shared Tailwind class tokens for consistent sizing
    ├── components/
    │   ├── SiteFooter.tsx
    │   └── SEO.tsx
    └── pages/
        ├── Landing.tsx         ← boot-sequence animation, skills matrix, publications, projects
        ├── About.tsx           ← extended bio and personal info
        ├── CVExport.tsx        ← full CV as downloadable PDF + HTML preview (/cv)
        ├── ResumeExport.tsx    ← condensed resume as downloadable PDF + HTML preview (/resume)
        └── Graph.tsx           ← interactive knowledge graph via Cytoscape (/graph)
```

## Routes

| Route | Description |
|-------|-------------|
| `/` | Landing — terminal boot animation, ranked skills matrix, publications, projects |
| `/about` | Extended bio, education, awards, links |
| `/cv` | Full CV — all publications grouped by status, all experience, downloadable PDF |
| `/resume` | Condensed resume — featured work, projects, skills, downloadable PDF |
| `/graph` | Interactive knowledge graph of skills, publications, projects, and experience |

## Development

```bash
npm install
npm run dev        # Vite dev server at localhost:5173
```

## Updating content

Edit `public/data/tyler-procko.yaml` only. The React app fetches it at runtime — no rebuild needed for content changes. The legacy HTML view requires `python3 build.py` to regenerate.

New publications are synced automatically every Monday via the OpenAlex workflow. Stubs are tagged `source: openalex` and `featured: false` for manual review.

## Building the legacy HTML view

```bash
pip install jinja2 pyyaml
python3 build.py   # writes public/legacy/index.html
```

## Deployment

Pushes to `main` trigger `.github/workflows/deploy.yml`, which:
1. Runs `build.py` to regenerate the legacy HTML from YAML
2. Runs `npm run build` to produce the Vite `dist/`
3. Deploys `dist/` to GitHub Pages at `procko.pro`

## History

The tag `pre-redesign` marks the last commit before the YAML-driven architecture was introduced. Prior to that, the site was a manually maintained static HTML file with no build system or data layer.
