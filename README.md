# procko.pro — Tyler T. Procko, Ph.D.

Personal portfolio and CV site for [Tyler T. Procko](https://procko.pro) — AI/Knowledge Graph Engineer.

## Architecture

This repo has one source of truth: **`data/tyler-procko.yaml`**. Everything else is generated or derived from it.

```
data/tyler-procko.yaml        ← single source of truth for all CV/portfolio content
│
├── build.py                  ← Python/Jinja2 builder → public/legacy/index.html
│   templates/                ← Jinja2 partials (one per section)
│   public/
│   ├── legacy/index.html     ← legacy HTML CV (generated by build.py, served at /legacy)
│   ├── data/tyler-procko.yaml← YAML copy served as a static asset for the React app
│   ├── CNAME                 ← custom domain (procko.pro)
│   └── 404.html              ← SPA routing redirect for GitHub Pages
│
└── src/                      ← React/Vite/TypeScript frontend
    ├── lib/yaml-loader.ts    ← fetches + validates YAML at runtime (Zod)
    ├── lib/schema.ts         ← Zod schema matching the YAML structure
    └── pages/
        ├── Landing.tsx       ← boot-sequence landing page, skills, publications, projects
        ├── Legacy.tsx        ← iframes the legacy HTML CV (/legacy)
        ├── CVExport.tsx      ← full CV as a generated PDF (/cv)
        ├── ResumeExport.tsx  ← condensed resume as a generated PDF (/resume)
        └── Graph.tsx         ← interactive knowledge graph via Cytoscape (/graph)
```

## Routes

| Route | Description |
|-------|-------------|
| `/` | Landing page — terminal boot animation, skills matrix, publications, projects |
| `/cv` | Full CV rendered as a downloadable PDF |
| `/resume` | Condensed resume rendered as a downloadable PDF |
| `/legacy` | Legacy HTML CV view (iframe), kept in sync with YAML |
| `/graph` | Interactive knowledge graph of skills, publications, and projects |

## Development

```bash
npm install
npm run dev        # Vite dev server
```

## Building the legacy HTML view

```bash
pip install jinja2 pyyaml
python3 build.py   # writes public/legacy/index.html
```

## Deployment

Pushes to `main` automatically trigger `.github/workflows/deploy.yml`, which:
1. Runs `build.py` to regenerate the legacy HTML from YAML
2. Runs `npm run build` to produce the Vite `dist/`
3. Deploys `dist/` to GitHub Pages (custom domain: `procko.pro`)

## Updating content

Edit `data/tyler-procko.yaml`. The React app reads it at runtime; the legacy HTML view is regenerated by running `build.py`. A CI parity test catches any drift between the YAML and the legacy HTML.

## History

The tag `pre-redesign` marks the last commit before this YAML-driven architecture was introduced. At that point the site was a manually maintained static HTML file with no build system or data layer.
